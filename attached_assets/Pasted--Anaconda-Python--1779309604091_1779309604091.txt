Окружающая среда Конда и обязательные пакеты

Anaconda - это дистрибутив Python для научных вычислений, целью которого является упрощение управления и развертывания пакетов.

Anaconda Server позволяет загружать, перемещать, копировать, делиться и загружать файл yaml в среде. Среда - это папка или каталог, который содержит определенную коллекцию пакетов конда и их зависимости. Это позволяет поддерживать их и работать отдельно без вмешательства друг друга.

Следуйте этому руководству, чтобы установить Anaconda или Miniconda на ваш ПК.

Создание и запуск веб-сервера Flask
Шаг 1: Создайте новый файл Python

Во-первых, создайте новый файл Python, названный app.pyв каталоге вашего проекта.

Если вы используете интерфейс терминала или командной строки, вы можете использовать touchКоманда для создания нового файла:

touch app.py

Или просто создать новый файл с именем app.pyв предпочитаемом тексте редакторе или интегрированной среде разработки (IDE).
Шаг 2: Начните писать код приложения Flask

Откройте app.pyфайл в текстовом редакторе или IDE.

    Импортная колба: Самый первый шаг - импорт класса Flask из модуля Flask.

    from flask import Flask

    Этот экземпляр класса Flask станет нашим приложением WSGI (Web Server Gateway Interface). WSGI - это спецификация, которая описывает, как веб-сервер взаимодействует с веб-приложениями. Для получения более подробной информации о WSGI, пожалуйста, перейдите по этой ссылке: WSGI Servers

    Создайте экземпляр класса Flask: Далее мы создадим экземпляр класса Flask, обычно называемый app. Это делается путем предоставления специальной переменной Python, __name__, который получил название текущего модуля. Это помогает Flak находить ресурсы, такие как шаблоны, которые могут понадобиться для веб-приложения.

    app = Flask(__name__)

    Зарегистрируйте маршрут: Маршрут в терминах веб-разработки - это шаблон URL, на который может ответить приложение. Для Flask мы можем определить, что наше приложение должно вернуть, когда запрашивается определенный URL, определяя маршрут и функцию. Мы используем декоратор, @app.route('/'), чтобы зарегистрировать функцию в качестве маршрута.

    @app.route('/')
    def home():
        return "Hello, World!"

    В этом примере восходящий слэш '/'Это самый простой маршрут, обычно представляющий домашнюю страницу веб-сайта. Когда кто-то посещает домашнюю страницу, функция homeБудет называться и "Здравствуйте, мир!" будет выставлена.

    Запустите приложение Flask: Наконец, чтобы запустить приложение Flask, нам нужно позвонить app.run. Обычно мы делаем это в рамках if __name__ == '__main__':блокировка для обеспечения запуска сервера только в том случае, если скрипт выполняется напрямую (не импортируется в качестве модуля).

    if __name__ == '__main__':
        app.run(host='0.0.0.0', debug=True)

    В вышеуказанной строке, host='0.0.0.0'позволяет вашему приложению быть доступным на любом IP, который есть у вашей машины. The debug=Trueаргумент обеспечивает режим отладки в Flask. В этом режиме сервер будет автоматически перезагружаться для изменений кода, и он предоставит полезную информацию об отладке, если произойдет ошибка.

После выполнения всех вышеперечисленных шагов ваше приложение Flask должно выглядеть так:

# Import the Flask class from the flask module
from flask import Flask

# Create an instance of the Flask class
app = Flask(__name__)

# Register a route
@app.route('/')
def home():
    return "Hello, World!"

# Run the Flask application
if __name__ == '__main__':
    app.run(host='0.0.0.0', debug=True)

Шаг 3: Запуск приложения Flask

Чтобы запустить приложение Flask, вам необходимо выполнить app.pyСценарий. Вы можете сделать это из терминала или командной строки со следующей командой:

python app.py

Теперь откройте веб-браузер и перейдите к своему локальному хосту на порту 5000 (обычно http://127.0.0.1:5000/или http://localhost:5000/) для просмотра вашего приложения Flask в действии.
